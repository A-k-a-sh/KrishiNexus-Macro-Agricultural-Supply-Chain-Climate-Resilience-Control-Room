# KrishiNexus Vercel Deployment Guide

This guide explains how to deploy the KrishiNexus fullstack application (React frontend + Express backend) to Vercel.

---

## Deployment Options

You can deploy this repository using two different setups:
- **Option A: Separate Projects (Recommended)**: Best for isolated builds, custom domains, and Vercel-native monorepo management.
- **Option B: Single Fullstack Project**: Best for simple single-domain configuration without setting up CORS or separate domains.

---

### Option A: Separate Projects (Recommended)

In this approach, you will import the repository twice into Vercel: once for the frontend React SPA, and once for the backend Express API.

#### 1. Backend Project Deployment
1. Go to the Vercel Dashboard and click **Add New** -> **Project**.
2. Connect your GitHub repository.
3. In the project configure settings:
   - **Project Name**: `krishinexus-backend` (or similar)
   - **Framework Preset**: `Other`
   - **Root Directory**: `backend`
4. Expand the **Environment Variables** section and add:
   - `MONGODB_URI`: Your MongoDB Atlas connection string.
   - `GEMINI_API_KEY`: Your Gemini API Key from Google AI Studio.
   - `CLIENT_URL`: The production URL of your frontend Vercel deployment (e.g. `https://krishinexus.vercel.app`).
   - `PORT`: `5001` (or leave empty)
   - `CRON_SECRET`: A long random secret key to secure your cron endpoint (e.g., `super-secret-key-123`).
5. Click **Deploy**. Vercel will automatically detect `backend/vercel.json` and build the serverless functions.
6. Note the deployment URL generated by Vercel for the backend (e.g., `https://krishinexus-backend.vercel.app`).

#### 2. Frontend Project Deployment
1. In the Vercel Dashboard, click **Add New** -> **Project** and import the same repository again.
2. In the project configure settings:
   - **Project Name**: `krishinexus`
   - **Framework Preset**: `Vite` (Vercel should auto-detect this)
   - **Root Directory**: `frontend`
3. Expand **Environment Variables** and add:
   - `VITE_API_URL`: The deployment URL of your backend project (e.g., `https://krishinexus-backend.vercel.app` - **without** a trailing slash).
4. Click **Deploy**. Vercel will build the frontend and host it.

---

### Option B: Single Fullstack Project

In this approach, you deploy the entire repository as a single Vercel project. The frontend and backend run on the same domain.

1. Go to the Vercel Dashboard and click **Add New** -> **Project**.
2. Import your GitHub repository.
3. In the project configure settings:
   - **Framework Preset**: `Other`
   - **Root Directory**: `.` (the repository root)
   - **Build Command**: Leave blank (Vercel will use the root `vercel.json` configuration to orchestrate builds)
4. Expand the **Environment Variables** section and add the backend variables:
   - `MONGODB_URI`: Your MongoDB Atlas connection string.
   - `GEMINI_API_KEY`: Your Gemini API Key from Google AI Studio.
   - `CRON_SECRET`: A long random secret key to secure your cron endpoint.
   - *Note: `VITE_API_URL` is NOT required here, as the frontend will fall back to calling `/api/...` on the same domain.*
5. Click **Deploy**. Vercel will build the React frontend and package the Express backend as serverless functions.

## Verifying Vercel Cron Jobs

Vercel Cron is configured inside the `vercel.json` configurations.
- The cron job is configured to hit `/api/cron/weather-refresh` every 6 hours (`0 */6 * * *`).
- To verify if it is registered:
  1. Open your backend/fullstack project page in the Vercel Dashboard.
  2. Click on the **Settings** tab.
  3. Look for **Cron Jobs** in the left sidebar.
  4. You should see the `/api/cron/weather-refresh` job scheduled.
  5. You can manually run it from the Vercel Dashboard by clicking the **Run** button to verify it updates weather and risk indicators correctly without timing out.

## MongoDB Atlas Configuration

For production deployment:
1. Ensure your MongoDB Atlas cluster network access allows connections from all IPs (`0.0.0.0/0`), since Vercel's serverless functions use dynamic IP addresses.
2. Verify that you have created the required MongoDB search indexes as specified in the project specifications document ([krishinexus_specs.md](file:///e:/Sciblitz%20project/krishiNexus-sicblitz%20cuet/krishiNexus-sicblitz%20cuet/others/krishinexus_specs.md#L1143-L1164)).
